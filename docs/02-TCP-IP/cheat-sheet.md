## 📊 Таблица уровней TCP/IP (суть в одном предложении)

| Уровень | Название | Ключевая идея | Пример команды |
|---------|----------|---------------|----------------|
| **1** | Связь | Кабель/Wi-Fi + MAC-адрес 🔌 | `ip link show` |
| **2** | Интернет | IP-адрес и маршруты 🌍 | `ip addr show` |
| **3** | Транспорт | Порты и пакеты (TCP/UDP) ✂️ | `netstat -tulpn` |
| **4** | Прикладной | Браузер/приложение 💻 | `curl https://...` |

## 🔑 Ключевые протоколы TCP/IP

### 📦 IP (Internet Protocol) — основа всего!
- **IPv4**: `192.168.1.10` (32 бита, 4 числа)
- **IPv6**: `2001:0db8:85a3:...` (128 бит, 8 групп)

### 🤝 TCP (Transmission Control Protocol) — надёжный!
- Гарантирует доставку всех пакетов
- Сортирует пакеты по порядку
- Пересылает потерянные пакеты
- **Используется**: Веб, почта, файлы

### 🚀 UDP (User Datagram Protocol) — быстрый!
- Не гарантирует доставку
- Быстрее TCP
- Потерянные пакеты не пересылаются
- **Используется**: Видеозвонки, онлайн-игры

## 🛠️ Команды для проверки каждого уровня (Linux/Mac)

```bash
# Уровень 1: Связь
ip link show eth0 | grep "state UP"

# Уровень 2: Интернет
ip addr show eth0 | grep inet
ip route show | grep default

# Уровень 3: Транспорт
netstat -tulpn | grep LISTEN

# Уровень 4: Прикладной
curl -v https://google.com | head -5
```

---

## 🛠️ Команды для проверки каждого уровня (Windows PowerShell)

```powershell
# Уровень 1: Связь
Get-NetAdapter -Name "Ethernet" | Select-Object Status, LinkSpeed

# Уровень 2: Интернет
Get-NetIPAddress -AddressFamily IPv4 | Select-Object IPAddress

# Уровень 3: Транспорт
Get-NetTCPConnection -State Listen | Select-Object LocalPort

# Уровень 4: Прикладной
curl -v https://google.com | head -5
```

## 🎯 Типичные проблемы и решения

### Проблема: "Не могу открыть сайт!"

| Уровень | Причина | Решение |
|---------|---------|---------|
| **1** | Нет Wi-Fi/кабеля | Включи сеть или подключи кабель 🔌 |
| **2** | Неправильный IP-адрес | Проверь `ip addr show` 🏠 |
| **3** | Порт заблокирован | Открой порт 80/443 на фаерволе ✂️ |
| **4** | DNS не работает | Попробуй публичный DNS (8.8.8.8) 🔍 |

## 🎬 Полный пример: Запрос к сайту в TCP/IP!

```
1. Ты вводишь google.com → Уровень 4 (браузер)
2. Данные шифруются HTTPS → Уровень 3 + 4
3. Устанавливается соединение → Уровень 3 (TCP)
4. Данные разбиваются на пакеты → Уровень 2 (IP)
5. Каждому пакету присваивается IP → Уровень 2 (маршрутизация)
6. Пакеты отправляются по кабелю/Wi-Fi → Уровень 1 (Ethernet/Wi-Fi)

Ответ идёт в обратном порядке! 🔄
```
