# 🗨️ Шпаргалка: Сессионный уровень (Session Layer)

## 🎯 Основные протоколы

### **RPC** — Remote Procedure Call
| Характеристика | Значение |
|----------------|----------|
| Назначение | Вызов удалённых функций как локальных |
| Использование | Микросервисы, облака |
| Надёжность | Через TCP (обычно) |

**Используется для:** Микросервисы, API, облачные приложения  
**Порт:** Разные (портmapper 111 + диапазон)

---

### **SPX** — Session Protocol Xerox
| Характеристика | Значение |
|----------------|----------|
| Назначение | Управление сессиями |
| Использование | NetWare (устарел) |
| Надёжность | Высокая |

**Используется для:** Устаревшие системы NetWare  
**Статус:** ❌ Почти не используется

---

### **NETBIOS** — Network Basic Input/Output System
| Характеристика | Значение |
|----------------|----------|
| Назначение | Локальные сети (печать, файлы) |
| Использование | Windows локальные сети |
| Надёжность | Средняя |

**Используется для:** Печать, файлообмен в Windows  
**Порты:** 137-139

---

## 🔧 Быстрые команды Linux

### Проверка RPC-сервисов:
```bash
# Показать все RPC-процессы
ps aux | grep rpc

# Показать работающие RPC-порты
rpcinfo -p localhost

# Или через systemctl
systemctl list-units --type=service | grep rpc
```

## 📊 Сравнение протоколов

| Протокол | Надёжность | Скорость | Использование | Статус |
|----------|------------|----------|---------------|--------|
| **RPC**  | Высокая (через TCP) | Средняя | Микросервисы, API | ✅ Активен |
| **SPX**  | Очень высокая | Низкая | NetWare | ❌ Устарел |
| **NETBIOS** | Средняя | Средняя | Локальные сети Windows | ⚠️ Локально |

## 🎯 Когда использовать что?

### ✅ RPC для:
- Микросервисов (вызов функций на других серверах)
- Облачных приложений (API вызовы)
- Удалённого доступа к данным
- Современных веб-приложений

### ✅ SPX для:
- ❌ Почти не используется (устарел)

### ✅ NETBIOS для:
- Локальных сетей Windows
- Печати в офисе
- Файлообмена между ПК
- Старых систем

## 🔄 Управление сессиями

### 3 шага работы сессии:

```
1. Установление → "Привет! Давай поговорим!"
2. Поддержание → "Я ещё здесь! ❤️" (каждые N секунд)
3. Завершение → "Разговор окончен! 👋"
```

## 📋 Основные порты сессионного уровня

| Порт | Протокол | Тип | Назначение |
|------|----------|-----|------------|
| 111  | RPC (portmapper) | TCP/UDP | Регистрация RPC-сервисов |
| 137-139 | NETBIOS | NetBIOS | Локальные сети Windows |

## 🛠️ Полезные команды

### Диагностика:
```bash
# Посмотреть работающие RPC-процессы
ps aux | grep rpc

# Проверить RPC-порты
rpcinfo -p localhost

# Показать NFS монтировки (используют RPC)
mount | grep nfs
```

### Настройка NFS:
```bash
# На сервере — разрешить доступ к папке
sudo nano /etc/exports
# Добавить: /home/shared 192.168.1.0/24(rw,sync,no_subtree_check)

# Применить изменения
sudo exportfs -ra

# На клиенте — смонтировать удалённую папку
sudo mount -t nfs server_ip:/home/shared /mnt/nfs
```

## 🎓 Ключевые отличия

### Сессионный vs Транспортный уровень:

| Характеристика | Сессионный (5) | Транспортный (4) |
|----------------|----------------|------------------|
| **Задача** | Управление диалогами | Надёжная доставка данных |
| **Протоколы** | RPC, SPX, NETBIOS | TCP, UDP |
| **Фокус** | Сессии между приложениями | Пакеты между хостами |

## 🎯 Контрольные точки (Checkpoints)

### Без контрольных точек:
```
Файл: 100 МБ
Сбой на 50 МБ → Передаём заново ВСЁ (100 МБ) 😱
```

### С контрольными точками:
```
Контрольные точки каждые 10 МБ
Сбой на 52 МБ → Передаём только [51-100] (49 МБ) ✅
```
