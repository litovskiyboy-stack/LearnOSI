## 1️⃣ Что такое IP-адрес? 📍

### Определение
**IP-адрес (Internet Protocol Address)** — это уникальный идентификатор устройства в сети, аналогичный домашнему адресу для письма.

### IPv4 vs IPv6

#### **IPv4** (старый стандарт)
- Формат: `192.168.1.10` (4 числа через точку)
- Размер: 32 бита = 4 байта
- Количество адресов: 2³² ≈ **4,3 миллиарда**
- Примеры:
  - `192.168.1.1` — домашний роутер (локальный)
  - `8.8.8.8` — Google DNS (публичный)

#### **IPv6** (новый стандарт)
- Формат: `2001:0db8:85a3:0000:0000:8a2e:0370:7334`
- Размер: 128 бит = 16 байт
- Количество адресов: 2¹²⁸ ≈ **340 триллионов триллионов**
- Примеры:
  - `fe80::1` — локальный адрес (link-local)

## 2️⃣ Что такое Gateway? 🌉

### Определение
**Gateway (шлюз)** — это устройство, которое соединяет две разные сети и позволяет им общаться друг с другом.

### Default Gateway (дефолтный шлюз)
- Адрес твоего роутера в локальной сети
- Все неизвестные адреса отправляются через него
- Пример: `192.168.1.1` или `10.0.0.1`

### Как проверить свой шлюз:

```bash
# Linux/Mac
ip route show | grep default

# Windows PowerShell
Get-NetRoute -AddressFamily IPv4 | Where-Object { $_.DestinationPrefix -eq "0.0.0.0/0" } | Select-Object NextHop
```

## 3️⃣ Что такое маска подсети? 🎭

### Определение
**Маска подсети (Subnet Mask)** — это "маска", которая разделяет IP-адрес на две части:
1. **Сеть** — к какой сети принадлежит устройство
2. **Хост** — конкретное устройство в этой сети

### Как работает маска:

#### Пример 1: Маска `255.255.255.0` (/24)
```
IP-адрес:    192.168.1.10 = 11000000.10101000.00000001.00001010
Маска:       255.255.255.0 = 11111111.11111111.11111111.00000000
---------------------------------------------------------------
Сеть:        192.168.1.x (первые 3 октета)
Хост:        .x (последний октет)

Всего адресов в сети: 2⁸ = 256
Используемых: 254 (исключая сеть и broadcast)
```

#### Пример 2: Маска `255.255.0.0` (/16)
```
IP-адрес:    192.168.x.x = 11000000.10101000.xxxxxxxx.xxxxxxxx
Маска:       255.255.0.0   = 11111111.11111111.00000000.00000000
---------------------------------------------------------------
Сеть:        192.168.x.x (первые 2 октета)
Хост:        .x.x (последние 2 октета)

Всего адресов в сети: 2¹⁶ = 65,536
Используемых: 65,534
```

### Как посчитать количество адресов:

**Формула:** `Количество адресов = 2^(32 - маска)`

| Маска | CIDR | Количество битов хоста | Адресов в сети | Используемых |
|-------|------|------------------------|----------------|--------------|
| 255.255.255.0 | /24 | 8 | 256 | 254 |
| 255.255.255.128 | /25 | 7 | 128 | 126 |
| 255.255.255.192 | /26 | 6 | 64 | 62 |
| 255.255.255.224 | /27 | 5 | 32 | 30 |

## 4️⃣ Как считать сети? 🧮

### Определение подсети:
Подсеть — это часть большого диапазона IP-адресов, которая выделяется отдельной организации или отделу.

### Пример расчёта подсети:

#### Задача: У тебя есть сеть `192.168.10.0/24`, нужно разделить на 4 подсети

**Шаг 1:** Определяем количество битов для подсетей
- Нужно 4 подсети = 2² = 4 → берём 2 бита из хоста

**Шаг 2:** Новый префикс
- Старый: /24, новый: /26 (добавляем 2 бита)

**Шаг 3:** Рассчитываем подсети
```
Подсеть 1: 192.168.10.0/26      → диапазон: 192.168.10.0 - 192.168.10.63
Подсеть 2: 192.168.10.64/26     → диапазон: 192.168.10.64 - 192.168.10.127
Подсеть 3: 192.168.10.128/26   → диапазон: 192.168.10.128 - 192.168.10.191
Подсеть 4: 192.168.10.192/26   → диапазон: 192.168.10.192 - 192.168.10.255
```

**Шаг 4:** Определяем важные адреса в каждой подсети
- **Network address**: первый адрес (например, `192.168.10.0`) — нельзя использовать
- **Broadcast address**: последний адрес (например, `192.168.10.63`) — нельзя использовать
- **Usable range**: адреса между ними (например, `192.168.10.1 - 192.168.10.62`)

### Как посчитать адрес подсети:

**Формула:** `Адрес подсети = IP & Маска` (битовое И)

#### Пример:
```
IP-адрес:    192.168.10.50 = 10100000.10101000.00001010.00110010
Маска:       255.255.255.0   = 11111111.11111111.11111111.00000000
---------------------------------------------------------------
Адрес подсети: 192.168.10.0 (первые 3 октета совпадают)
```

### Как посчитать broadcast-адрес:

**Формула:** `Broadcast = Адрес подсети + (2^h - 1)` где h — битов хоста

#### Пример:
```
IP-адрес:    192.168.10.50/24
Маска:       /24 → 8 битов хоста
Адрес подсети: 192.168.10.0
Broadcast:   192.168.10.0 + (2⁸ - 1) = 192.168.10.255
```

## 5️⃣ Какие типы сетей бывают? 🌐

### По масштабу:

#### **LAN (Local Area Network)** — локальная сеть
- Небольшая территория: дом, офис, здание
- Скорость: 10-1000 Mbps (Gigabit Ethernet)
- Примеры:
  - Домашняя Wi-Fi сеть
  - Офисная сеть с коммутатором

#### **MAN (Metropolitan Area Network)** — городская сеть
- Город или регион
- Скорость: 10-100 Mbps
- Примеры:
  - Сеть университета в городе

#### **WAN (Wide Area Network)** — глобальная сеть
- Страна, континент, мир
- Скорость: 1-100 Mbps (зависит от расстояния)
- Примеры:
  - Интернет (самая большая WAN!)

#### **PAN (Personal Area Network)** — персональная сеть
- Личное пространство человека
- Скорость: до 10 Mbps
- Примеры:
  - Bluetooth между телефоном и наушниками

### По технологии:

#### **Wired (проводная)**
- Ethernet (кабель)
- Fiber optic (оптика)
- Coaxial (коаксиальный кабель)

#### **Wireless (беспроводная)**
- Wi-Fi (802.11 a/b/g/n/ac/ax/be)
- Bluetooth
- Cellular (4G/LTE, 5G)

## 6️⃣ Что такое NAT? 🔄

### Определение
**NAT (Network Address Translation)** — перевод сетевых адресов. Позволяет нескольким устройствам с локальным IP выходить в интернет через один публичный IP! 🌐

### Зачем нужен NAT:
1. **Экономия IP-адресов** — одному роутеру достаточно 1 публичного IP для всех устройств
2. **Безопасность** — скрытие внутренней сети от внешнего мира
3. **Удобство** — не нужно менять IP на каждом устройстве

### Как работает NAT:

#### Пример без NAT (проблема):
```
Домашняя сеть: 192.168.1.x (локальные адреса)
Провайдер даёт: 1 публичный IP = 1 устройство может быть в интернете!
```

#### Решение с NAT:
```
Твой компьютер (192.168.1.10) → Роутер (NAT) → Интернет (публичный IP роутера)
                                    ↓
Роутер запоминает: "Компьютер 192.168.1.10 хочет на google.com"
И возвращает ответ обратно этому компьютеру!
```

### Типы NAT:

#### **Full Cone NAT** — полный конус
- Любое устройство может отвечать на запрос из интернета
- Самый простой, но наименее безопасный

#### **Restricted Cone NAT** — ограниченный конус
- Устройство может отвечать только тем, кто запрашивал его
- Более безопасный

### Как проверить работу NAT:

```bash
# Посмотреть таблицу трансляций (Linux)
cat /proc/net/nf_nat

# Проверить публичный IP
curl ifconfig.me
```

## 7️⃣ Что такое DHCP? 📡

### Определение
**DHCP (Dynamic Host Configuration Protocol)** — автоматическая выдача IP-адресов устройствам в сети! 🎁

### Зачем нужен DHCP:
1. **Автоматизация** — не нужно вручную настраивать IP на каждом устройстве
2. **Экономия адресов** — адреса можно перераспределять
3. **Удобство** — подключил устройство, и оно сразу в сети!

### Как работает DHCP (4 шага):

#### **DORA процесс:**

1. **D**iscover — клиент ищет сервер
   ```
   Client: "Есть кто-нибудь DHCP-сервер?!" (broadcast)
   ```

2. **O**ffer — сервер предлагает адрес
   ```
   Server: "Вот тебе IP 192.168.1.100, используй его!"
   ```

3. **R**equest — клиент просит этот адрес
   ```
   Client: "Хорошо, беру 192.168.1.100!"
   ```

4. **A**cknowledge — сервер подтверждает
   ```
   Server: "Согласен, IP 192.168.1.100 твой до 24 часов"
   ```

### Пример DHCP в действии:

```bash
# Посмотреть текущий IP (полученный через DHCP)
ip addr show eth0 | grep inet

# Проверить DHCP-сервер (если настроен)
netstat -tulpn | grep :67  # Порт 67 — DHCP сервер
```

## 8️⃣ Что такое DNS? 🔍

### Определение
**DNS (Domain Name System)** — телефонная книга интернета! Переводит доменные имена в IP-адреса. 📞

### Зачем нужен DNS:
1. **Удобство** — помним `google.com`, а не `142.250.x.x`
2. **Гибкость** — можно изменить IP сайта, не меняя домен
3. **Нагрузка** — разные IP для разных регионов

### Как работает DNS:

#### Пример запроса к google.com:

```
Ты вводишь: google.com
    ↓
Браузер спрашивает у DNS-сервера: "Какой IP у google.com?"
    ↓
DNS-сервер ищет в своей базе (или спрашивает у других серверов)
    ↓
Получает ответ: 142.250.x.x
    ↓
Браузер открывает сайт по этому IP!
```

### Типы DNS-записей:

#### **A Record** — адресная запись
```
google.com → 142.250.x.x (IPv4)
```

#### **AAAA Record** — адресная запись для IPv6
```
google.com → 2607:f8b0:4004... (IPv6)
```

#### **CNAME Record** — алиас (псевдоним)
```
www.google.com → google.com
```

#### **MX Record** — почтовый сервер
```
gmail.com → mail.gmail.com (порт 25)
```

### Как проверить работу DNS:

```bash
# Проверить разрешение домена
nslookup google.com

# Или через dig (более подробный)
dig google.com +short

# Посмотреть кэш DNS (Linux/Mac)
cat /etc/resolv.conf
```

### Популярные DNS-серверы:

| Сервер | Адрес | Особенности |
|--------|-------|-------------|
| Google DNS | 8.8.8.8, 8.8.4.4 | Быстрый, надёжный |
| Cloudflare DNS | 1.1.1.1, 1.0.0.1 | Самый быстрый, приватный |

## 9️⃣ Какие протоколы существуют? 📋

### Основные сетевые протоколы:

#### **TCP/IP Suite** — основа интернета!

| Протокол | Порт | Назначение |
|----------|------|------------|
| **HTTP** | 80 | Веб-сайты (незащищённый) |
| **HTTPS** | 443 | Веб-сайты (зашифрованный) |
| **FTP** | 21 | Передача файлов |
| **SSH** | 22 | Удалённый доступ к серверу |
| **SMTP** | 25, 587 | Отправка почты |
| **DNS** | 53 | Перевод доменов в IP |

#### **Transport Layer Protocols:**

| Протокол | Особенности | Используется для |
|----------|-------------|------------------|
| **TCP** | Надёжный, гарантирует доставку | Веб, почта, файлы |
| **UDP** | Быстрый, без гарантий | Видеозвонки, онлайн-игры |

#### **Application Layer Protocols:**

| Протокол | Назначение | Примеры |
|----------|------------|---------|
| **HTTP/HTTPS** | Веб-трафик | Браузеры, API |
| **FTP/SFTP** | Передача файлов | FileZilla, scp |
| **SMTP/POP3/IMAP** | Почта | Gmail, Outlook |

## 🔟 Какие типы интернета есть? 🌍

### По типу доступа:

#### **Public Internet (публичный интернет)**
- Открытая сеть для всех
- Доступ через провайдера
- Пример: домашний Wi-Fi, мобильный интернет

#### **Private Network (частная сеть)**
- Закрытая сеть организации
- Доступ только по авторизации
- Пример: корпоративная сеть банка

#### **Intranet (внутренняя сеть)**
- Частная сеть с веб-серверами
- Доступ внутри организации
- Пример: внутренний портал компании

### По технологии:

#### **Broadband (высокоскоростной интернет)**
- DSL (ADSL/VDSL) — через телефонную линию
- Cable — через кабельное TV
- Fiber optic — оптоволокно (самый быстрый!)
- 4G/5G — мобильный интернет
