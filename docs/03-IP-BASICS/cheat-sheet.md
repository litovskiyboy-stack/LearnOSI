# 📋 Шпаргалка: IP-адресация и сетевые протоколы

## 🌐 IP-адреса

### IPv4 формат: `192.168.1.10` (32 бита)
```
Октет 1 . Октет 2 . Октет 3 . Октет 4
  192       .   168      .    1     .   10
```

### IPv6 формат: `2001:db8::1` (128 бит)

## 🎭 Маски подсети и CIDR

| CIDR | Маска | Адресов в сети | Используемых |
|------|-------|----------------|--------------|
| /8   | 255.0.0.0    | 16,777,216     | 16,777,214  |
| /16  | 255.255.0.0  | 65,536         | 65,534      |
| /24  | 255.255.255.0| 256            | 254         |
| /25  | 255.255.255.128| 128           | 126         |
| /26  | 255.255.255.192| 64            | 62          |
| /27  | 255.255.255.224| 32            | 30          |
| /28  | 255.255.255.240| 16            | 14          |

| /30  | 255.255.255.252| 4             | 2 (PPP)     |

**Формула:** `Адресов = 2^(32 - маска)`

## 🌉 Gateway (Шлюз)
- **Что это?** Адрес твоего роутера
- **Как проверить?** `ip route show | grep default`
- **Пример:** `192.168.1.1`

---

## 🔄 NAT (Network Address Translation)
- **Зачем?** Экономия IP, безопасность
- **Как работает?** Роутер переводит локальные IP в публичный
- **Проверить:** `curl ifconfig.me` vs локальный IP

---

## 📡 DHCP (Dynamic Host Configuration Protocol)
- **Зачем?** Автоматическая выдача IP
- **4 шага:** DORA (Discover, Offer, Request, Acknowledge)
- **Порт сервера:** 67

---

## 🔍 DNS (Domain Name System)
- **Зачем?** Перевод доменов в IP (телефонная книга интернета!)
- **Как проверить?** `nslookup google.com` или `dig google.com +short`
- **Популярные серверы:**
  - Google: 8.8.8.8, 8.8.4.4
  - Cloudflare: 1.1.1.1, 1.0.0.1

## 🧮 Быстрый расчёт подсетей

### Пример: `192.168.10.0/26`
```
Маска: /26 = 255.255.255.192
Диапазон: 192.168.10.0 - 192.168.10.63
Используемых адресов: 62 (64 - 2)

Network:   192.168.10.0 (нельзя использовать)
First host: 192.168.10.1
Last host:  192.168.10.62
Broadcast:  192.168.10.63 (нельзя использовать)
```

### Формулы:
- **Адрес подсети:** `IP & Маска` (битовое И)
- **Broadcast:** `Адрес подсети + (2^h - 1)` где h — битов хоста

## 🛠️ Полезные команды Linux

```bash
# IP-адрес
ip addr show eth0 | grep inet

# Gateway
ip route show | grep default

# Маска подсети
ip addr show eth0 | grep 'inet\b' | awk '{print $2}' | cut -d/ -f1

# DNS серверы
cat /etc/resolv.conf

# Публичный IP
curl ifconfig.me

# Проверка сети
ping -c 4 google.com

# Открытые порты
netstat -tulpn | grep LISTEN

# Сетевой трафик в реальном времени
watch -n 1 'netstat -i'

# Анализ трафика (требует root)
sudo tcpdump -i eth0 port 80 -nn
```

## 🎯 Ключевые отличия

### TCP vs UDP
| Характеристика | TCP | UDP |
|----------------|-----|-----|
| Надёжность | Гарантирует доставку | Без гарантий |
| Скорость | Медленнее (проверки) | Быстрее |
| Использование | Веб, почта, файлы | Видеозвонки, игры |

### LAN vs WAN
| Характеристика | LAN | WAN |
|----------------|-----|-----|
| Масштаб | Маленький | Огромный |
| Скорость | 10-1000 Mbps | 1-100 Mbps |
| Пример | Домашняя сеть | Интернет |
