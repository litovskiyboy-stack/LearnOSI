# 🔐 Полное руководство по слою 6: Представление (Presentation Layer)

## Введение

**Представительский уровень** — это "переводчик и шифровальщик" интернета! Он отвечает за то, чтобы данные были понятны разным системам и безопасны при передаче. 

### Основные задачи представительства:
1. **Шифрование (Encryption)** — защита данных от перехвата
2. **Сжатие (Compression)** — уменьшение размера файлов для быстрой передачи
3. **Кодирование (Encoding)** — перевод данных в понятный формат
4. **Валидация (Validation)** — проверка целостности данных

---

## 1️⃣ Шифрование данных: Подробный разбор

### 🔐 Что такое шифрование?

**Шифрование** — это процесс преобразования читаемых данных в нечитаемый вид, который может быть расшифрован только авторизованным получателем.

```
Исходные данные (plaintext): "Привет! Это секрет!"
   ↓ [Шифрование с ключом]
Зашифрованные данные (ciphertext): "xK9#mP2$vL8@nQ4" (непонятный набор символов)
   ↓ [Расшифровка с тем же ключом]
Исходные данные: "Привет! Это секрет!" ✅
```

### 🗝️ Типы шифрования:

#### 1. Симметричное шифрование (Symmetric Encryption)

**Как работает:** Используется один и тот же ключ для шифрования и расшифровки.

```
Ключ: "secret123"

Отправитель → [Шифровать с ключом] → Зашифрованные данные
Получатель ← [Расшифровать с тем же ключом] ← Зашифрованные данные
```

**Преимущества:**
- Быстрое (меньше вычислительных затрат)
- Простое в реализации
- Хорошо подходит для больших объёмов данных

**Недостатки:**
- Нужно безопасно передать ключ отправителю и получателю
- Если ключ скомпрометирован — все данные уязвимы

**Пример алгоритмов:**
- **AES (Advanced Encryption Standard)** — современный стандарт
- **DES (Data Encryption Standard)** — старый, но ещё используется
- **3DES** — улучшенная версия DES

#### 2. Асимметричное шифрование (Asymmetric Encryption)

**Как работает:** Используется пара ключей: открытый (public key) и закрытый (private key).

```
Отправитель → [Шифровать с открытым ключом] → Зашифрованные данные
Получатель ← [Расшифровать с закрытым ключом] ← Зашифрованные данные
```

**Преимущества:**
- Безопаснее (не нужно передавать секретный ключ)
- Позволяет цифровые подписи
- Идеально для обмена ключами

**Недостатки:**
- Медленнее симметричного шифрования
- Требует больше вычислительных ресурсов

**Пример алгоритмов:**
- **RSA (Rivest–Shamir–Adleman)** — самый популярный
- **ECC (Elliptic Curve Cryptography)** — современный, эффективный
- **DSA (Digital Signature Algorithm)** — для цифровых подписей

### 🔄 Гибридное шифрование:

**Как работает:** Комбинация симметричного и асимметричного шифрования!

```
1. Генерируем случайный ключ для симметричного шифрования
2. Шифруем данные с этим ключом (быстро)
3. Шифруем сам ключ с помощью открытого ключа получателя (безопасно)
4. Отправляем: [зашифрованные данные] + [зашифрованный ключ]

Получатель:
1. Расшифровывает ключ своим закрытым ключом
2. Расшифровывает данные с этим ключом ✅
```

**Пример использования:** HTTPS, PGP для email

## 2️⃣ Сжатие данных: Подробный разбор

### 📦 Что такое сжатие?

**Сжатие данных** — это процесс уменьшения размера файлов путём удаления избыточной информации.

```
Исходный файл: 10 МБ (текст, картинки)
   ↓ [Сжатие ZIP]
Сжатый файл: 2 МБ (в 5 раз меньше!)
   ↓ [Распаковка получателем]
Исходный файл: 10 МБ ✅
```

### 🎯 Типы сжатия:

#### 1. Сжатие без потерь (Lossless Compression)

**Как работает:** Все данные сохраняются, только удаляются избыточные паттерны.

**Примеры алгоритмов:**
- **ZIP/DEFLATE** — стандарт для архивов
- **GZIP** — сжатие файлов и HTTP-ответов
- **LZ77/LZ78** — старые, но эффективные алгоритмы

```
Текст: "Привет! Привет! Привет!"
         ↑      ↑      ↑
       Повторяющиеся слова

Сжатый текст: "Привет!\n[ПОВТОРИТЬ 2 раза]"
# Вместо 3 раз — один раз + указание повторить! ✅
```

**Использование:**
- Текстовые файлы (код, документы)
- Архивы программ
- Базы данных

#### 2. Сжатие с потерями (Lossy Compression)

**Как работает:** Удаляется часть данных, которые незаметны для человека или не важны.

```
Картинка: [1000 пикселей с одинаковым цветом]
         ↑
       Избыточная информация

Сжатая картинка: [1 пиксель цвета + "повторить 999 раз"]
# Вместо 1000 пикселей — один! ✅
```

**Использование:**
- Фотографии (JPEG)
- Аудио (MP3, AAC)
- Видео (H.264, H.265)

### 🛠️ Алгоритмы сжатия:

#### 1. DEFLATE (Data Deflate Algorithm)

**Как работает:** Комбинация LZ77 и Huffman кодирования!

```
Шаг 1: LZ77 — поиск повторяющихся паттернов
   Текст: "Привет! Привет! Привет!"
           ↑      ↑      ↑
         Повторяющиеся слова
   
   Результат: "Привет!\n[ПОВТОРИТЬ 2 раза]"

Шаг 2: Huffman кодирование — замена частых символов на короткие коды
   '!' встречается часто → короткий код (например, "0")
   'Пр' встречается редко → длинный код (например, "1010101")

Итоговый результат: ещё больше сжатие! ✅
```

**Использование:** ZIP, GZIP, PNG

#### 2. JPEG (Joint Photographic Experts Group)

**Как работает:** Удаление избыточной информации в изображениях!

```
Шаг 1: Разделение на блоки 8x8 пикселей
   Картинка → [БЛОК 1] [БЛОК 2] ... [БЛОК N]

Шаг 2: Преобразование ДКП (Discrete Cosine Transform)
   Каждый блок → частотные компоненты
   Низкие частоты (важные) + Высокие частоты (мелкие детали)

Шаг 3: Квантование — удаление высоких частот
   Высокие частоты → уменьшаем точность (теряем незаметно)

Шаг 4: Сжатие DEFLATE
   Удаляем повторяющиеся паттерны ✅
```

**Использование:** Фотографии в интернете, социальные сети

#### 3. MP3 (MPEG-1 Audio Layer III)

**Как работает:** Удаление звуков, которые не слышит человек!

```
Шаг 1: Анализ частотного спектра
   Аудио → [Низкие частоты] [Средние частоты] [Высокие частоты]

Шаг 2: Психологическая модель слуха
   Если есть громкий звук на одной частоте — тихие звуки рядом не слышны!
   
Шаг 3: Удаление "незаметных" звуков
   Тихие звуки рядом с громкими → удаляем ✅

Шаг 4: Сжатие DEFLATE
   Удаляем повторяющиеся паттерны ✅
```

**Использование:** Музыка в интернете, подкасты

## 3️⃣ Кодирование данных: Подробный разбор

### 🔄 Что такое кодирование?

**Кодирование данных** — это процесс перевода данных из одного формата в другой для передачи или хранения.

```
Исходные данные (ASCII): "Привет!" (непонятно для русских букв 😱)
   ↓ [UTF-8 кодировка]
Закодированные данные (UTF-8): "Привет!" (понятно всем! ✅)
```

### 📊 Типы кодирования:

#### 1. ASCII → UTF-8

**ASCII (American Standard Code for Information Interchange):**
- Кодировка для английских букв и символов
- Только 7 бит на символ (128 символов)
- Не поддерживает русские буквы! 😱

```
ASCII: "Привет!" → [НЕПОНЯТНО] ❌
# Русские буквы не в ASCII!
```

**UTF-8 (Unicode Transformation Format - 8-bit):**
- Кодировка для всех языков мира
- Поддерживает миллионы символов
- Обратима к ASCII для английских букв ✅

```
UTF-8: "Привет!" → [ПОНИМАЕТСЯ ВСЕМ] ✅
# Русские буквы в UTF-8!
```

**Как работает UTF-8:**
```
Английская буква 'A' → 1 байт (0x41)
Русская буква 'П' → 2 байта (0xD0 0x9F)
Кириллическая буква 'Р' → 2 байта (0xD0 0xA0)

Итог: UTF-8 использует больше байтов для не-английских символов! ✅
```

#### 2. Бинарные данные → Base64

**Base64 кодирование:**
- Перевод бинарных данных в текстовый формат
- Используется для передачи бинарных файлов по email, HTTP и т.д.

```
Бинарные данные: 10101010... (непонятный набор битов 😱)
   ↓ [Base64 кодировка]
Base64 текст: "YWJjZGVm..." (понятный текст! ✅)
```

**Как работает Base64:**
```
Шаг 1: Разбиваем бинарные данные на группы по 3 байта (24 бита)
   Данные: [0x41][0x42][0x43] = "ABC"

Шаг 2: Переводим в Base64 алфавит (A-Z, a-z, 0-9, +, /)
   24 бита → 4 символа из Base64
   
   Пример: [0x41][0x42][0x43] = "YWJj"

Шаг 3: Добавляем '=' для заполнения (если нужно)
   Если данных меньше 3 байт — добавляем 1 или 2 '='
```

**Использование:**
- Передача изображений по email
- JWT токены в веб-приложениях
- Base64 в URL-параметрах

#### 3. HTML → XML/JSON

**HTML (HyperText Markup Language):**
- Для отображения страниц в браузере
- Не подходит для обмена данными между системами

```html
<html>
  <body>
    <h1>Привет!</h1>
  </body>
</html>
```

**JSON (JavaScript Object Notation):**
- Для обмена данными между API
- Простой и понятный формат

```json
{
  "title": "Привет!",
  "content": "Это заголовок"
}
```

## 4️⃣ Ключевые протоколы: Подробный разбор

### 🔐 SSL/TLS — Secure Sockets Layer / Transport Layer Security

#### Что это?

**SSL/TLS** — протокол для шифрования данных в интернете! Используется в HTTPS, защищённых email и т.д.

```
HTTPS = HTTP + SSL/TLS (шифрование) 🔐
HTTP  = незащищённый веб ❌
HTTPS = защищённый веб ✅
```

#### Как работает TLS?

**Шаг 1: Установление соединения (Handshake)**

```
Клиент → Сервер: "Привет! Я хочу зашифрованный разговор!"
   ↓ [Client Hello]
Сервер → Клиент: "Я готов! Вот мой сертификат + список поддерживаемых алгоритмов"
   ↓ [Server Hello + Certificate]
Клиент → Сервер: "Проверяю твой сертификат..."
   ↓ [Certificate Verification]
Клиент → Сервер: "Сертификат проверен! Начинаем шифрование!"
   ↓ [Key Exchange]
```

**Шаг 2: Генерация ключей**

```
Клиент и сервер генерируют случайный ключ для симметричного шифрования!
# Этот ключ используется только один раз ✅
```

**Шаг 3: Шифрование данных**

```
Клиент → Сервер: [ЗАШИФРОВАННЫЕ ДАННЫЕ 🔐]
Сервер → Клиент: [ЗАШИФРОВАННЫЙ ОТВЕТ 🔐]
# Только получатель может прочитать! ✅
```

#### Пример кода TLS в Python:

```python
import ssl
import socket

# Создание SSL контекста
context = ssl.create_default_context()
context.check_hostname = True  # Проверять доменное имя
context.verify_mode = ssl.CERT_REQUIRED  # Требовать сертификат

# Подключение к серверу с HTTPS
with socket.create_connection(('example.com', 443)) as sock:
    with context.wrap_socket(sock, server_hostname='example.com') as ssock:
        print(f"Сервер: {ssock.getpeercert()}")  # Показать сертификат
        
        # Передача данных (зашифрованные)
        data = b"Привет! Это зашифрованное сообщение!"
        ssock.sendall(data)
        
        # Получение ответа (зашифрованный)
        response = ssock.recv(1024).decode()
        print(f"Ответ: {response}")
```

#### Преимущества TLS:

1. **Конфиденциальность:** Данные шифруются, их не могут перехватить
2. **Целостность:** Данные не были изменены в пути
3. **Аутентификация:** Сервер подтверждён сертификатом (не подделка)

#### Недостатки TLS:

1. **Производительность:** Handshake замедляет соединение
2. **Сложность:** Нужно правильно настроить сертификаты
3. **Стоимость:** Купленные доменные сертификаты стоят денег

### 📸 JPEG — Joint Photographic Experts Group

#### Что это?

**JPEG** — формат для сжатия фотографий! Используется в 90% изображений в интернете!

```
Исходная фото: 5 МБ (все детали)
   ↓ [JPEG сжатие]
JPEG файл: 500 КБ (в 10 раз меньше!)
   ↓ [Распаковка]
Фото: почти как оригинал, но в 10 раз легче! ✅
```

#### Как работает JPEG?

**Шаг 1: Разделение на блоки 8x8 пикселей**

```
Картинка → [БЛОК 1] [БЛОК 2] ... [БЛОК N]
# Каждый блок 8x8 = 64 пикселя
```

**Шаг 2: Преобразование ДКП (Discrete Cosine Transform)**

```
Каждый блок → частотные компоненты
Низкие частоты (важные) + Высокие частоты (мелкие детали)
```

**Шаг 3: Квантование — удаление высоких частот**

```
Высокие частоты → уменьшаем точность (теряем незаметно)
# Человеческий глаз не видит мелкие детали! ✅
```

**Шаг 4: Сжатие DEFLATE**

```
Удаляем повторяющиеся паттерны ✅
```

#### Пример кода JPEG в Python:

```python
from PIL import Image
import numpy as np

# Открыть изображение
img = Image.open('original.png')
img_array = np.array(img)

# Преобразование в RGB (если нужно)
if img.mode != 'RGB':
    img = img.convert('RGB')
    img_array = np.array(img)

# Сохранить как JPEG с разным качеством
for quality in [50, 75, 100]:
    img.save(f'output_quality_{quality}.jpg', 'JPEG', quality=quality)
    
    # Показать размер файла
    size = os.path.getsize(f'output_quality_{quality}.jpg')
    print(f"Качество {quality}: {size / 1024:.2f} КБ")

# Сравнение качества
original_size = os.path.getsize('original.png')
print(f"\nОригинал: {original_size / 1024:.2f} КБ")
```

#### Преимущества JPEG:

1. **Малый размер:** В 5-20 раз меньше оригинала
2. **Широкая поддержка:** Поддерживается всеми браузерами и приложениями
3. **Качество:** При высоком качестве почти неотличимо от оригинала

#### Недостатки JPEG:

1. **Сжатие с потерями:** Часть данных теряется навсегда
2. **Артефакты:** При сильном сжатии появляются "квадратики"
3. **Не подходит для текста/графики:** Лучше использовать PNG или SVG

### 📦 ZIP — Zip File Format

#### Что это?

**ZIP** — формат для сжатия файлов и архивов! Используется для распространения программ, резервного копирования и т.д.

```
Исходные файлы: 10 МБ (5 разных файлов)
   ↓ [ZIP архивация]
ZIP архив: 2 МБ (в 5 раз меньше!)
   ↓ [Распаковка]
Файлы: как были, но в 5 раз быстрее загрузка! ✅
```

#### Как работает ZIP?

**Шаг 1: Сжатие каждого файла отдельно**

```
Файл 1 → [СЖАТЬ DEFLATE] → Сжатый файл 1
Файл 2 → [СЖАТЬ DEFLATE] → Сжатый файл 2
...
Файл N → [СЖАТЬ DEFLATE] → Сжатый файл N
```

**Шаг 2: Создание индекса**

```
[Список файлов в архиве + метаданные]
# Название, размер, дата создания и т.д.
```

**Шаг 3: Добавление заголовка**

```
[Заголовок ZIP-архива] + [Сжатые файлы] + [Индекс]
```

#### Пример кода ZIP в Python:

```python
import zipfile
import os

# Создание ZIP-архива
with zipfile.ZipFile('archive.zip', 'w', zipfile.ZIP_DEFLATED) as zipf:
    # Добавляем файлы
    for file in ['file1.txt', 'file2.txt']:
        if os.path.exists(file):
            zipf.write(file, arcname=file)

# Распаковка ZIP-архива
with zipfile.ZipFile('archive.zip', 'r') as zipf:
    # Показать содержимое
    print("Содержимое архива:")
    for info in zipf.infolist():
        size = info.file_size
        compressed = info.compress_size
        ratio = 1 - (compressed / size) if size > 0 else 0
        
        print(f"{info.filename}: {size} байт → {compressed} байт ({ratio*100:.1f}% сжатие)")
    
    # Распаковать все файлы
    zipf.extractall(path='/tmp/')

# Добавление файлов в существующий архив
with zipfile.ZipFile('archive.zip', 'a') as zipf:
    zipf.write('new_file.txt', arcname='new_file.txt')
```

#### Преимущества ZIP:

1. **Без потерь:** Все данные сохраняются
2. **Универсальность:** Поддерживается всеми ОС и приложениями
3. **Гибкость:** Можно добавлять/удалять файлы без распаковки всего архива

#### Недостатки ZIP:

1. **Медленное сжатие:** DEFLATE требует времени для больших файлов
2. **Нет поддержки потоковой передачи:** Нужно скачать весь файл перед распаковкой

## 5️⃣ Практические примеры использования

### Пример 1: HTTPS соединение (SSL/TLS)

#### Что происходит при посещении защищённого сайта?

```
1. Установление соединения (TCP порт 443)
   Клиент → Сервер: "Привет! Я хочу зашифрованный разговор!"
   
2. Handshake (обмен ключами)
   Клиент → Сервер: "Вот мой сертификат"
   Сервер → Клиент: "Вот мой сертификат + открытый ключ"
   
3. Проверка сертификатов
   Клиент проверяет:
   - Выдан ли сертификат доверенным центром?
   - Не истёк ли срок действия?
   - Подходит ли для этого домена?
   
4. Шифрование соединения
   Клиент → Сервер: "Соединение зашифровано! Начинаем!"
   
5. Передача данных (зашифрованная)
   Клиент → Сервер: [ЗАШИФРОВАННЫЕ ДАННЫЕ 🔐]
   Сервер → Клиент: [ЗАШИФРОВАННЫЙ ОТВЕТ 🔐]
   
6. Завершение соединения
   Клиент → Сервер: "Разговор окончен!"
```

#### Почему HTTPS безопасен?

**Без HTTPS:**
```
Клиент → Сервер: "Мой пароль: secret123"
# Данные передаются в открытом виде! 😱
Хакер перехватывает: "Мой пароль: secret123" ❌
```

**С HTTPS:**

### Пример 2: Создание ZIP-архива с Python

#### Код для создания архива программ:

```python
import zipfile
import os
from datetime import datetime

def create_program_archive(program_name, output_path):
    """Создание ZIP-архива программы"""
    
    # Создаём временную папку для сборки
    temp_dir = f'/tmp/{program_name}_{datetime.now().strftime("%Y%m%d_%H%M%S")}'
    os.makedirs(temp_dir)
    
    # Копируем файлы в временную папку
    files_to_archive = [
        'main.py',
        'requirements.txt',
        'README.md',
        'config.json'
    ]
    
    for file in files_to_archive:
        if os.path.exists(file):
            with open(file, 'rb') as src:
                with open(os.path.join(temp_dir, file), 'wb') as dst:
                    dst.write(src.read())
    
    # Создаём ZIP-архив
    archive_path = output_path.replace('.zip', f'_{datetime.now().strftime("%Y%m%d_%H%M%S")}.zip')
    
    with zipfile.ZipFile(archive_path, 'w', zipfile.ZIP_DEFLATED) as zipf:
        for root, dirs, files in os.walk(temp_dir):
            for file in files:
                file_path = os.path.join(root, file)
                arcname = os.path.relpath(file_path, temp_dir)
                
                # Добавляем файл в архив
                info = zipfile.ZipInfo(arcname)
                zipf.writestr(info, open(file_path, 'rb').read())
    
    print(f"Архив создан: {archive_path}")
    print(f"Размер оригинала: {sum(os.path.getsize(f) for f in files_to_archive if os.path.exists(f)) / 1024:.2f} КБ")
    print(f"Размер архива: {os.path.getsize(archive_path) / 1024:.2f} КБ")

# Использование
create_program_archive('my_app', 'my_app.zip')
```

### Пример 3: Конвертация изображений в JPEG с Python

#### Код для оптимизации фотографий:

```python
from PIL import Image
import os

def optimize_images(input_dir, output_dir):
    """Оптимизация изображений в JPEG"""
    
    # Создаём выходную папку
    os.makedirs(output_dir, exist_ok=True)
    
    for filename in os.listdir(input_dir):
        if not filename.lower().endswith(('.png', '.jpg', '.jpeg')):
            continue
        
        input_path = os.path.join(input_dir, filename)
        
        # Открываем изображение
        img = Image.open(input_path)
        
        # Конвертируем в RGB (если нужно)
        if img.mode != 'RGB':
            img = img.convert('RGB')
        
        # Оптимизируем с разным качеством
        for quality in [85, 90, 100]:
            output_path = os.path.join(output_dir, f"{filename}_q{quality}.jpg")
            
            # Сохраняем как JPEG
            img.save(output_path, 'JPEG', quality=quality, optimize=True)
            
            print(f"Создан: {output_path}")
            print(f"  Размер: {os.path.getsize(output_path) / 1024:.2f} КБ")

# Использование
optimize_images('input_photos/', 'optimized_photos/')
```

## 🛠️ Настройка SSL/TLS на веб-сервере (Apache):

### Установка сертификата:

```bash
# Установить Apache и OpenSSL
sudo apt install apache2 openssl

# Генерация самоподписанного сертификата
openssl req -x509 -nodes -days 365 -newkey rsa:2048 \
  -keyout /etc/ssl/private/apache-selfsigned.key \
  -out /etc/ssl/certs/apache-selfsigned.crt

# Включить SSL в Apache
sudo nano /etc/apache2/sites-available/000-default.conf

# Добавить:
<VirtualHost *:443>
    SSLEngine on
    SSLCertificateFile /etc/ssl/certs/apache-selfsigned.crt
    SSLCertificateKeyFile /etc/ssl/private/apache-selfsigned.key
    
    # Настройки шифрования
    SSLProtocol all -SSLv2 -SSLv3
    SSLCipherSuite HIGH:!aNULL:!MD5
</VirtualHost>

# Включить модули SSL
sudo a2enmod ssl
sudo systemctl restart apache2
```
