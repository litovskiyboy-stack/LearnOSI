# 🔐 Слой 6: Представление (Presentation Layer)

## Введение

**Представительский уровень** — это "переводчик и шифровальщик" интернета! Он отвечает за то, чтобы данные были понятны разным системам и безопасны при передаче. Представь, что ты говоришь на русском, а собеседник — на английском: тебе нужен переводчик (кодирование) и секретарь-шифровальщик для конфиденциальных разговоров! 🌐🔒

---

## 🎯 Основные задачи представительства:

1. **Шифрование** — защита данных от перехвата
2. **Сжатие** — уменьшение размера файлов для быстрой передачи
3. **Кодирование** — перевод данных в понятный формат
4. **Валидация** — проверка целостности данных

---

## 🔐 Шифрование (Encryption)

### Что это?
Преобразование данных в нечитаемый вид, чтобы только получатель мог их прочитать!

```
Исходные данные: "Привет! Это секрет!"
   ↓ [Шифрование]
Зашифрованные данные: "xK9#mP2$vL8@nQ4" (непонятный набор символов)
   ↓ [Расшифровка получателем]
Исходные данные: "Привет! Это секрет!" ✅
```

### Как работает шифрование?

**Симметричное шифрование:**
- Один ключ для шифрования и расшифровки
- Быстрое, но нужно безопасно передать ключ

```
Ключ: "secret123"

Отправитель → [Шифровать с ключом] → Зашифрованные данные
Получатель ← [Расшифровать с тем же ключом] ← Зашифрованные данные
```

**Асимметричное шифрование:**
- Два ключа: открытый (для шифрования) и закрытый (для расшифровки)
- Безопаснее, но медленнее

```
Отправитель → [Шифровать с открытым ключом] → Зашифрованные данные
Получатель ← [Расшифровать с закрытым ключом] ← Зашифрованные данные
```

## 📦 Сжатие данных (Data Compression)

### Что это?
Уменьшение размера файлов для быстрой передачи!

```
Исходный файл: 10 МБ (текст, картинки)
   ↓ [Сжатие ZIP]
Сжатый файл: 2 МБ (в 5 раз меньше!)
   ↓ [Распаковка получателем]
Исходный файл: 10 МБ ✅
```

### Как работает сжатие?

**Удаление повторяющихся данных:**
```
Текст: "Привет! Привет! Привет!"
         ↑      ↑      ↑
       Повторяющиеся слова

Сжатый текст: "Привет!\n[ПОВТОРИТЬ 2 раза]"
# Вместо 3 раз — один раз + указание повторить! ✅
```

**Удаление избыточной информации:**
```
Картинка: [1000 пикселей с одинаковым цветом]
         ↑
       Избыточная информация

Сжатая картинка: [1 пиксель цвета + "повторить 999 раз"]
# Вместо 1000 пикселей — один! ✅
```

## 🔄 Кодирование данных (Data Encoding)

### Что это?
Перевод данных в понятный формат для передачи!

### Примеры кодирования:

#### ASCII → UTF-8
```
ASCII: "Привет!" (непонятно для русских букв 😱)
   ↓ [UTF-8 кодировка]
UTF-8: "Привет!" (понятно всем! ✅)
```

#### Бинарные данные → Base64
```
Бинарные данные: 10101010... (непонятный набор битов 😱)
   ↓ [Base64 кодировка]
Base64: "YWJjZGVm..." (понятный текст! ✅)
```

## 🛠️ Ключевые протоколы и форматы:

### SSL/TLS — Secure Sockets Layer / Transport Layer Security

**Что это?** Протокол для шифрования данных в интернете! 🔐

**Как работает:**
1. **Установление соединения (Handshake)**
   - Клиент и сервер обмениваются ключами
   - Проверяют сертификаты (чтобы не подделка)

2. **Шифрование данных**
   - Все данные шифруются перед отправкой
   - Только получатель может их прочитать

3. **Завершение соединения**
   - Проверка целостности данных
   - Закрытие соединения

```
Клиент → Сервер: "Привет! Я хочу зашифрованный разговор!"
   ↓ [Handshake]
Сервер → Клиент: "Я готов! Вот мой сертификат (подпись)"
   ↓ [Проверка сертификата]
Клиент → Сервер: "Сертификат проверен! Начинаем шифрование!"
   ↓ [Шифрование данных]
Клиент → Сервер: [ЗАШИФРОВАННЫЕ ДАННЫЕ 🔐]
Получатель ← [Расшифровка] ← Зашифрованные данные ✅
```

**Пример использования:**
- HTTPS (защищённый веб)
- SSH (защищённый удалённый доступ)
- Email (SMTPS, IMAPS)

### JPEG — Joint Photographic Experts Group

**Что это?** Формат для сжатия фотографий! 📸

**Как работает:**
1. **Удаление избыточной информации**
   - Человеческий глаз не видит мелкие детали
   - Убираем то, что незаметно

2. **Квантование**
   - Группируем похожие цвета
   - Уменьшаем количество цветов

3. **Сжатие с потерями**
   - Часть данных теряется (но незаметно)
   - Файл становится в 10-20 раз меньше!

```
Исходная фото: 5 МБ (все детали)
   ↓ [JPEG сжатие]
JPEG файл: 500 КБ (в 10 раз меньше!)
   ↓ [Распаковка]
Фото: почти как оригинал, но в 10 раз легче! ✅
```

**Пример использования:**
- Фотографии в интернете
- Социальные сети
- Мобильные приложения

### ZIP — Zip File Format

**Что это?** Формат для сжатия файлов и архивов! 📦

**Как работает:**
1. **Сжатие каждого файла отдельно**
   - Каждый файл сжимается независимо

2. **Удаление повторяющихся данных**
   - Ищем повторяющиеся паттерны

3. **Создание индекса**
   - Список файлов в архиве

```
Исходные файлы: 10 МБ (5 разных файлов)
   ↓ [ZIP архивация]
ZIP архив: 2 МБ (в 5 раз меньше!)
   ↓ [Распаковка]
Файлы: как были, но в 5 раз быстрее загрузка! ✅
```

**Пример использования:**
- Распространение программ
- Резервное копирование
- Передача файлов по email

**Что это?** Форматы для обмена данными между системами! 📊

- Широкая поддержка в старых системах

## 🔍 Практический пример: HTTPS соединение

### Что происходит при посещении защищённого сайта?

```
1. Установление соединения (TCP порт 443)
   Клиент → Сервер: "Привет! Я хочу зашифрованный разговор!"
   
2. Handshake (обмен ключами)
   Клиент → Сервер: "Вот мой сертификат"
   Сервер → Клиент: "Вот мой сертификат + открытый ключ"
   
3. Проверка сертификатов
   Клиент проверяет:
   - Выдан ли сертификат доверенным центром?
   - Не истёк ли срок действия?
   - Подходит ли для этого домена?
   
4. Шифрование соединения
   Клиент → Сервер: "Соединение зашифровано! Начинаем!"
   
5. Передача данных (зашифрованная)
   Клиент → Сервер: [ЗАШИФРОВАННЫЕ ДАННЫЕ 🔐]
   Сервер → Клиент: [ЗАШИФРОВАННЫЙ ОТВЕТ 🔐]
   
6. Завершение соединения
   Клиент → Сервер: "Разговор окончен!"
```

### Почему HTTPS безопасен?

**Без HTTPS:**
```
Клиент → Сервер: "Мой пароль: secret123"
# Данные передаются в открытом виде! 😱
Хакер перехватывает: "Мой пароль: secret123" ❌
```

**С HTTPS:**
```
Клиент → Сервер: [ЗАШИФРОВАННЫЕ ДАННЫЕ 🔐]
# Только сервер может прочитать! ✅
Хакер видит: [НЕПОНЯТНЫЙ НАБОР СИМВОЛОВ] ✅
```

## 🛠️ Настройка SSL/TLS на веб-сервере (Apache):

### Установка сертификата:

```bash
# Установить Apache и OpenSSL
sudo apt install apache2 openssl

# Генерация самоподписанного сертификата
openssl req -x509 -nodes -days 365 -newkey rsa:2048 \
  -keyout /etc/ssl/private/apache-selfsigned.key \
  -out /etc/ssl/certs/apache-selfsigned.crt

# Включить SSL в Apache
sudo nano /etc/apache2/sites-available/000-default.conf

# Добавить:
<VirtualHost *:443>
    SSLEngine on
    SSLCertificateFile /etc/ssl/certs/apache-selfsigned.crt
    SSLCertificateKeyFile /etc/ssl/private/apache-selfsigned.key
</VirtualHost>

# Включить модули SSL
sudo a2enmod ssl
sudo systemctl restart apache2
```

### Проверка работы HTTPS:

```bash
# Проверить сертификат
openssl s_client -connect localhost:443 -showcerts

# Или через браузер (должно быть 🔒)
https://localhost/
```

## 🛠️ Создание ZIP-архива с Linux:

### Сжатие файлов в ZIP:

```bash
# Создать ZIP-архив из нескольких файлов
zip -r archive.zip file1.txt file2.txt folder1/

# Добавить файлы в существующий архив
zip -u archive.zip new_file.txt

# Показать содержимое архива без распаковки
unzip -l archive.zip

# Распаковать архив
unzip archive.zip

# Сжать с максимальным уровнем (медленнее, но меньше размер)
zip -9 archive.zip file1.txt file2.txt
```

### Пример кода ZIP-архивации в Python:

```python
import zipfile
import os

# Создание ZIP-архива
with zipfile.ZipFile('archive.zip', 'w', zipfile.ZIP_DEFLATED) as zipf:
    # Добавляем файлы
    for file in ['file1.txt', 'file2.txt']:
        if os.path.exists(file):
            zipf.write(file, arcname=file)

# Распаковка ZIP-архива
with zipfile.ZipFile('archive.zip', 'r') as zipf:
    # Показать содержимое
    print(zipf.namelist())
    
    # Распаковать все файлы
    zipf.extractall(path='/tmp/')
```

## 🛠️ Настройка сжатия изображений (JPEG):

### Конвертация в JPEG с Python:

```python
from PIL import Image

# Открыть изображение
img = Image.open('original.png')

# Сохранить как JPEG (сжатие с потерями)
img.save('compressed.jpg', 'JPEG', quality=85)  # качество 85%

# Или с максимальным качеством
img.save('best_quality.jpg', 'JPEG', quality=100)
```

### Командная строка для конвертации:

```bash
# Конвертировать PNG в JPEG
convert input.png output.jpg

# Сжать JPEG (уменьшить качество)
convert input.jpg -quality 75 output_compressed.jpg

# Изменить размер изображения
convert input.jpg -resize 800x600 output_resized.jpg
```
