# 🌉 Шпаргалка: Транспортный уровень (Transport Layer)

## 🎯 Основные протоколы

### **TCP** — Transmission Control Protocol
| Характеристика | Значение |
|----------------|----------|
| Надёжность | ✅ Гарантирует доставку |
| Скорость | ⚡ Средняя (проверки) |
| Установка соединения | Да (3-way handshake) |
| Порядок пакетов | ✅ Сохраняет порядок |

**Используется для:** Веб-сайты, почта, файлы, SSH  
**Порты:** 80 (HTTP), 443 (HTTPS), 21 (FTP), 22 (SSH)

---

### **UDP** — User Datagram Protocol
| Характеристика | Значение |
|----------------|----------|
| Надёжность | ❌ Без гарантий |
| Скорость | 🚀 Быстрая |
| Установка соединения | Нет |
| Порядок пакетов | ❌ Не гарантирует |

**Используется для:** Видеозвонки, онлайн-игры, DNS  
**Порты:** 53 (DNS), 67/68 (DHCP), 123 (NTP)

---

## 🔧 Быстрые команды Linux

### Проверка TCP-портов:
```bash
# Открытые порты (TCP)
netstat -tulpn | grep LISTEN

# Все соединения
netstat -an | grep ESTABLISHED

# Прослушать порт
nc -l -p 8080
```

### Проверка UDP-портов:
```bash
# Открытые порты (UDP)
ss -ulpn

# Проверить UDP-порт
nc -u -zv google.com 53
```

## 📊 Сравнение TCP vs UDP

| Характеристика | TCP | UDP |
|----------------|-----|-----|
| Надёжность | Высокая | Низкая |
| Скорость | Средняя | Высокая |
| Установка соединения | Да | Нет |
| Порядок пакетов | Да | Нет |
| Перегрузка сети | Контролируется | Без контроля |

## 🎯 Когда использовать что?

### ✅ TCP для:
- Веб-сайтов (HTTP/HTTPS)
- Почты (SMTP, IMAP, POP3)
- Файлов (FTP, SFTP)
- Удалённого доступа (SSH)
- Баз данных

### ✅ UDP для:
- Видеозвонков (Zoom, Skype)
- Онлайн-игр
- Стриминга видео
- DNS запросов
- DHCP выдачи IP

## 🔍 3-way handshake (TCP)

```
1. Клиент → Сервер: SYN "Привет!"
2. Сервер → Клиент: SYN-ACK "Я слышу!"
3. Клиент → Сервер: ACK "Согласен!"
Результат: Соединение установлено! 🎉
```

## 📋 Основные порты транспортного уровня

| Порт | Протокол | Тип | Назначение |
|------|----------|-----|------------|
| 80   | HTTP     | TCP | Веб-сайты (незащищённый) |
| 443  | HTTPS    | TCP | Веб-сайты (зашифрованный) |
| 21   | FTP      | TCP | Передача файлов |
| 22   | SSH      | TCP | Удалённый доступ |
| 25, 587 | SMTP  | TCP | Отправка почты |
| 53   | DNS      | UDP | Перевод доменов в IP |
| 67, 68 | DHCP  | UDP | Выдача IP-адресов |
| 123  | NTP      | UDP | Синхронизация времени |

## 🛠️ Полезные команды

### Диагностика:
```bash
# Посмотреть все TCP-параметры
sysctl net.ipv4.tcp_*

# Количество активных соединений
netstat -an | grep ESTABLISHED | wc -l

# Проверить, открыт ли порт
telnet google.com 80
# или
curl -v https://google.com
```

### Настройка:
```bash
# Увеличить время ожидания закрытого соединения
sudo sysctl -w net.ipv4.tcp_fin_timeout=30

# Включить SYN cookies (защита от DDoS)
sudo sysctl -w net.ipv4.tcp_syncookies=1
```

## 🎓 Ключевые отличия

### TCP — как заказное письмо:
- Гарантирует доставку ✅
- Можно отследить статус ✅
- Медленнее (проверки) ⏱️
- Идеально для важных данных 💼

### UDP — как открытка:
- Быстро и просто 🚀
- Может потеряться ❌
- Не отслеживается ❌
- Идеально для неважных данных 🎮
