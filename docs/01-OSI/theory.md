# 🌐 Модель OSI (Open Systems Interconnection)

## Введение

Модель OSI — это концептуальная рамка для понимания того, как работают компьютерные сети. Представь её как **почтовую систему**: письмо проходит через множество этапов, прежде чем дойти до адресата! 📮

---

## 7 уровней OSI (от простого к сложному)

### 1️⃣ Физический уровень (Physical Layer)
**Аналогия:** Бумага, ручка, почтовый ящик

- **Что делает:** Передаёт биты (0 и 1) по физическим каналам
- **Примеры:** Ethernet-кабели, Wi-Fi сигналы, оптоволокно
- **Ключевые технологии:**
  - IEEE 802.3 (Ethernet)
  - IEEE 802.11 (Wi-Fi)
  - RS-232 (последовательные порты)

```bash
# Пример: проверка физического соединения
ping -c 4 192.168.1.1
# Если нет ответа — проблема на физическом уровне!
```

---

### 2️⃣ Канальный уровень (Data Link Layer)
**Аналогия:** Печать конверта с MAC-адресами "от кого" и "кому"

- **Что делает:** Добавляет MAC-адреса, проверяет ошибки на коротком участке сети
- **Примеры:** Ethernet-кабель между твоим ПК и роутером
- **Ключевые технологии:**
  - IEEE 802.3 (Ethernet)
  - VLAN (Virtual LAN)
  - Switching

```bash
# Пример: просмотр MAC-адресов соседей
arp -a
# Показывает таблицу ARP — сопоставление IP и MAC адресов
```

---

### 3️⃣ Сетевой уровень (Network Layer)
**Аналогия:** Почтовый индекс и город в адресе

- **Что делает:** Определяет путь через всю сеть, IP-адресация
- **Примеры:** Твой IP `192.168.1.10` → роутер → интернет
- **Ключевые технологии:**
  - IPv4/IPv6 (IP-адресация)
  - Routing (маршрутизация)
  - ICMP (ping, traceroute)

```bash
# Пример: просмотр своей IP-конфигурации
ip addr show
# Или в Windows: ipconfig /all
```

---

### 4️⃣ Транспортный уровень (Transport Layer)
**Аналогия:** Упаковка письма в коробку с гарантией доставки

- **Что делает:** Гарантирует доставку, разбивает на сегменты, TCP/UDP
- **Примеры:** TCP гарантирует, что все части письма дойдут; UDP — быстро, но без гарантий
- **Ключевые технологии:**
  - TCP (Transmission Control Protocol) — надёжный
  - UDP (User Datagram Protocol) — быстрый

```bash
# Пример: просмотр открытых портов и протоколов
netstat -tulpn
# Показывает, какие программы слушают на каких портах
```

---

### 5️⃣ Сессионный уровень (Session Layer)
**Аналогия:** Звонок другу: "Привет, давай поговорим"

- **Что делает:** Устанавливает, поддерживает и завершает сеансы связи между приложениями
- **Ключевые функции:**
  - **Handshake/Handoff**: установление соединения перед передачей данных
  - **Keep-alive**: поддержание активного соединения (ping-pong пакеты)
  - **Checkpointing**: сохранение позиции в передаче больших файлов
  - **Multiplexing**: несколько разговоров одновременно через одно соединение

**Примеры из жизни:**
1. Когда ты открываешь сайт — браузер устанавливает SSL-сессию с сервером
2. FTP: контроль-соединение отдельно от данных
3. Telnet/SSH: постоянная сессия до закрытия терминала

**Ключевые технологии:**
- **NetBIOS**: старые Windows сети (имена хостов)
- **RPC (Remote Procedure Call)**: вызов функций на удалённом компьютере
- **HTTP Keep-Alive**: повторное использование TCP соединения для нескольких запросов

```bash
# Пример: просмотр активных сессий и соединений
netstat -an | grep ESTABLISHED
# Показывает все активные соединения (сессии)

# Проверка HTTP Keep-Alive
curl -v https://google.com --keepalive-time 60
```

**Типичные проблемы:**
- Сессия "зависла" — нужно перезапустить соединение
- Таймаут keep-alive слишком короткий → постоянные переподключения
- Конфликт сессионных ID в многопоточных приложениях

---

### 6️⃣ Представленческий уровень (Presentation Layer)
**Аналогия:** Перевод письма с русского на английский + упаковка в конверт

- **Что делает:** Преобразование данных между прикладным уровнем и сетью. "Переводчик" и "упаковщик"! 📦🌐
- **Ключевые функции:**
  - **Шифрование**: защита данных от перехвата (SSL/TLS, PGP)
  - **Сжатие**: уменьшение размера данных (ZIP, GZIP, JPEG)
  - **Конвертация форматов**: PNG ↔ JPEG, PDF ↔ HTML
  - **Кодирование**: ASCII ↔ UTF-8, EBCDIC ↔ ASCII

**Примеры из жизни:**
1. Когда ты отправляешь email — текст шифруется SSL/TLS перед отправкой
2. Браузер конвертирует HTML/CSS в отрисовку на экране
3. ZIP-архив сжимает файлы перед загрузкой

**Ключевые технологии:**
- **SSL/TLS**: HTTPS, Secure Sockets Layer (порты 443, 8443)
- **PGP/GPG**: шифрование email и файлов
- **JPEG/PNG/GIF/WEBP**: форматы изображений с разной степенью сжатия
- **MP3/MPEG/AAC**: аудиоформаты
- **ZIP/GZIP/BZIP2**: архивация и сжатие
- **PDF/HTML/XML**: форматы документов

```bash
# Пример 1: проверка SSL сертификата сайта
openssl s_client -connect google.com:443 -showcerts | openssl x509 -text

# Пример 2: просмотр заголовков HTTP (Presentation layer в действии)
curl -v https://google.com 2>&1 | grep "HTTP/2"

# Пример 3: сжатие файла GZIP
gzip myfile.txt
gunzip myfile.txt.gz

# Пример 4: конвертация изображения
convert input.jpg output.png  # требует ImageMagick
```

**Типичные проблемы:**
- Неподдерживаемый формат (браузер не понимает новый видеоформат)
- SSL сертификат просрочен или неверный → ошибка безопасности в браузере
- Сжатие слишком сильное → медленная загрузка при плохом интернете

---

### 7️⃣ Прикладной уровень (Application Layer)
**Аналогия:** Само письмо с текстом "Дорогой друг, как дела?" 📝💌

- **Что делает:** Прямой интерфейс для пользователя или приложения. Всё, что ты видишь и используешь!
- **Ключевые функции:**
  - Предоставление сетевых сервисов пользователям
  - Определение формата данных (JSON, XML, HTML)
  - Обработка пользовательского ввода
  - Генерация отчётов и уведомлений

**Примеры из жизни:**
1. Браузер Chrome/Firefox/Safari — ты видишь сайты здесь! 🌐
2. Почтовая программа Outlook/Gmail — отправляешь письма здесь
3. Telegram/WhatsApp/Viber — мессенджеры работают на этом уровне
4. FTP клиент для загрузки файлов
5. SSH терминал для удалённого доступа

**Ключевые протоколы и порты:**

| Протокол | Порт | Назначение | Пример использования |
|----------|------|------------|---------------------|
| **HTTP/HTTPS** | 80 / 443 | Веб-браузинг | `google.com`, `youtube.com` |
| **SMTP** | 25, 587, 465 | Отправка email | Gmail SMTP сервер |
| **POP3/IMAP** | 110 / 993 | Получение email | Outlook почта |
| **FTP/SFTP** | 20-21 / 22 | Передача файлов | Загрузка архивов |
| **DNS** | 53 | Перевод доменов в IP | `google.com` → `142.250.x.x` |
| **SSH** | 22 | Удалённый доступ | Подключение к серверу |
| **Telnet** | 23 | Текстовый терминал | ❌ небезопасно! |
| **DNS over HTTPS (DoH)** | 443 | Безопасный DNS | Cloudflare DNS |

```bash
# Примеры работы с прикладными протоколами:

# HTTP запрос с подробным выводом
curl -v https://google.com | head -20

# SMTP проверка (порт 25)
telnet mail.google.com 25
# Должно показать: "220 Gmail SMTP ready"

# DNS lookup (перевод домена в IP)
dig google.com +short
nslookup google.com

# FTP анонимный вход
ftp anonymous@ftp.example.com
# Логин: guest, Пароль: твоя email

# SSH подключение к удалённому серверу
ssh user@example.com  # порт по умолчанию 22

# Проверка открытых портов на компьютере
netstat -tulpn | grep LISTEN
```

**Типичные проблемы:**
- Сайт не открывается → проблема на уровне 7 (браузер, DNS) или ниже
- Email не отправляется → SMTP сервер недоступен или блокировка провайдера
- FTP не загружает файлы → права доступа или firewall блокирует порт 21/22

## 📊 Сравнение OSI и TCP/IP

| Уровень OSI | Соответствует TCP/IP | Примеры протоколов |
|-------------|---------------------|-------------------|
| 7 (Application) | Application | HTTP, FTP, SMTP, DNS |
| 6 (Presentation) | Application | SSL/TLS, JPEG, PNG |
| 5 (Session) | Application | Session management, RPC |
| 4 (Transport) | Transport | TCP, UDP, SCTP |
| 3 (Network) | Internet | IP, ICMP, ARP, IGMP |
| 2 (Data Link) | Network Interface | Ethernet, Wi-Fi, PPP |
| 1 (Physical) | Network Interface | Cables, signals, radio waves |

**Важно:** TCP/IP имеет меньше уровней — уровни 5-7 OSI объединены в один Application уровень!

```bash
# Физический (уровень 1) - есть ли кабель/Wi-Fi?

# Сетевой (уровень 3) - твоя IP?

# Транспортный (уровень 4) - какие порты открыты?

```

- Почему HTTPS безопаснее HTTP? (Шифрование на уровне 6)
- Что такое "Keep-Alive" и зачем он нужен? (Уровень 5 - экономия соединений)
