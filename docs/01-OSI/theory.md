# 🌐 Модель OSI (Open Systems Interconnection)

## Введение

Модель OSI — это концептуальная рамка для понимания того, как работают компьютерные сети. Представь её как **почтовую систему**: письмо проходит через множество этапов, прежде чем дойти до адресата! 📮

---

## 7 уровней OSI (от простого к сложному)

### 1️⃣ Физический уровень (Physical Layer)
**Аналогия:** Бумага, ручка, почтовый ящик

- **Что делает:** Передаёт биты (0 и 1) по физическим каналам
- **Примеры:** Ethernet-кабели, Wi-Fi сигналы, оптоволокно
- **Ключевые технологии:**
  - IEEE 802.3 (Ethernet)
  - IEEE 802.11 (Wi-Fi)
  - RS-232 (последовательные порты)

```bash
# Пример: проверка физического соединения
ping -c 4 192.168.1.1
# Если нет ответа — проблема на физическом уровне!
```

---

### 2️⃣ Канальный уровень (Data Link Layer)
**Аналогия:** Печать конверта с MAC-адресами "от кого" и "кому"

- **Что делает:** Добавляет MAC-адреса, проверяет ошибки на коротком участке сети
- **Примеры:** Ethernet-кабель между твоим ПК и роутером
- **Ключевые технологии:**
  - IEEE 802.3 (Ethernet)
  - VLAN (Virtual LAN)
  - Switching

```bash
# Пример: просмотр MAC-адресов соседей
arp -a
# Показывает таблицу ARP — сопоставление IP и MAC адресов
```

---

### 3️⃣ Сетевой уровень (Network Layer)
**Аналогия:** Почтовый индекс и город в адресе

- **Что делает:** Определяет путь через всю сеть, IP-адресация
- **Примеры:** Твой IP `192.168.1.10` → роутер → интернет
- **Ключевые технологии:**
  - IPv4/IPv6 (IP-адресация)
  - Routing (маршрутизация)
  - ICMP (ping, traceroute)

```bash
# Пример: просмотр своей IP-конфигурации
ip addr show
# Или в Windows: ipconfig /all
```

---

### 4️⃣ Транспортный уровень (Transport Layer)
**Аналогия:** Упаковка письма в коробку с гарантией доставки

- **Что делает:** Гарантирует доставку, разбивает на сегменты, TCP/UDP
- **Примеры:** TCP гарантирует, что все части письма дойдут; UDP — быстро, но без гарантий
- **Ключевые технологии:**
  - TCP (Transmission Control Protocol) — надёжный
  - UDP (User Datagram Protocol) — быстрый

```bash
# Пример: просмотр открытых портов и протоколов
netstat -tulpn
# Показывает, какие программы слушают на каких портах
```

---

### 5️⃣ Сессионный уровень (Session Layer)
**Аналогия:** Звонок другу: "Привет, давай поговорим"

- **Что делает:** Устанавливает и разрывает соединения
- **Примеры:** Handshake при подключении к сайту
- **Ключевые технологии:**
  - Session establishment/termination
  - Keep-alive механизмы

---

### 6️⃣ Представленческий уровень (Presentation Layer)
**Аналогия:** Перевод письма с русского на английский

- **Что делает:** Шифрование, сжатие, форматы данных
- **Примеры:** SSL/TLS шифрование, JPEG/PNG конвертация
- **Ключевые технологии:**
  - SSL/TLS (шифрование)
  - JPEG, PNG, GIF (форматы изображений)
  - ZIP, GZIP (сжатие)

```bash
# Пример: проверка SSL сертификата сайта
openssl s_client -connect google.com:443 -showcerts
```

---

### 7️⃣ Прикладной уровень (Application Layer)
**Аналогия:** Само письмо с текстом "Дорогой друг..."

- **Что делает:** Интерфейс для пользователя
- **Примеры:** Браузер Chrome, почтовая программа Outlook
- **Ключевые технологии:**
  - HTTP/HTTPS (веб)
  - SMTP/POP3/IMAP (почта)
  - FTP/SFTP (файлы)

```bash
# Пример: просмотр HTTP-запроса в браузере
curl -v https://google.com
# Показывает весь процесс запроса и ответа
```

---

## 🔄 Жизненный пример: Загрузка сайта Google

Когда ты вводишь `google.com` в браузере:

1. **Прикладной:** Браузер готовит запрос "покажи мне Google" (HTTP)
2. **Представленческий:** Шифрует запрос HTTPS, конвертирует в HTTP/2
3. **Сессионный:** Устанавливает SSL-соединение с сервером
4. **Транспортный:** TCP разбивает на сегменты и гарантирует доставку
5. **Сетевой:** IP-пакеты идут через роутеры к Google (142.250.x.x)
6. **Канальный:** Ethernet-кабель передаёт пакеты между устройствами
7. **Физический:** Электрические импульсы в кабеле или радиоволны Wi-Fi

## 📊 Сравнение OSI и TCP/IP

| Уровень OSI | Соответствует TCP/IP | Примеры протоколов |
|-------------|---------------------|-------------------|
| 7 (Application) | Application | HTTP, FTP, SMTP |
| 6 (Presentation) | Application | SSL/TLS, JPEG |
| 5 (Session) | Application | Session management |
| 4 (Transport) | Transport | TCP, UDP |
| 3 (Network) | Internet | IP, ICMP, ARP |
| 2 (Data Link) | Network Interface | Ethernet, Wi-Fi |
| 1 (Physical) | Network Interface | Cables, signals |
