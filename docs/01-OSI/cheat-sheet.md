## 📊 Таблица уровней OSI (суть в одном предложении)

| Уровень | Название | Ключевая идея | Пример команды |
|---------|----------|---------------|----------------|
| **1** | Физический | Электричество/свет в проводе ⚡ | `ethtool eth0` |
| **2** | Канальный | MAC-адрес внутри сети 📮 | `ip link show` |
| **3** | Сетевой | IP-адрес между сетями 🏠 | `ip addr show` |
| **4** | Транспортный | Порты и пакеты ✂️ | `netstat -tulpn` |
| **5** | Сессионный | Установить связь 📞 | `netstat -an` |
| **6** | Представленческий | Шифрование/форматы 🔒 | `curl -v https://...` |
| **7** | Прикладной | Браузер/приложение 🌐 | `lsof -i` |

## 🔑 Ключевые термины (чтобы не путаться!)

### Физический уровень:
- **Бит**: 0 или 1 (единица информации)
- **Сигнал**: электрический/оптический импульс
- **Протоколы**: Ethernet, USB, Wi-Fi

### Канальный уровень:
- **MAC-адрес**: `AA:BB:CC:DD:EE:FF` (паспорт устройства)
- **Фрейм**: пакет данных с адресами
- **ARP**: узнаёт MAC по IP

### Сетевой уровень:
- **IP-адрес**: `192.168.1.10` (логический адрес)
- **Маршрутизация**: как найти путь к получателю
- **Подсеть**: часть большой сети

### Транспортный уровень:
- **TCP**: надёжный, с гарантией доставки 📦✅
- **UDP**: быстрый, без гарантий 🚀
- **Порт**: номер приложения (80=HTTP, 443=HTTPS)

### Сессионный уровень:
- **Соединение**: установление связи перед передачей
- **Синхронизация**: деление данных на части
- **Многоканальность**: несколько разговоров одновременно

### Представленческий уровень:
- **Шифрование**: HTTPS, SSL/TLS 🔒
- **Сжатие**: ZIP, GZIP 📦
- **Форматы**: JPEG, MP3, PDF 🖼️🎵📄

### Прикладной уровень:
- **Браузеры**: Chrome, Firefox 🌐
- **Почта**: SMTP, POP3, IMAP 📧
- **Мессенджеры**: WhatsApp, Telegram 💬

## 🛠️ Команды для проверки каждого уровня (Linux/Mac)

```bash
# Уровень 1: Физический
ethtool eth0 | grep "Link detected"

# Уровень 2: Канальный
ip link show eth0 | grep link/ether

# Уровень 3: Сетевой
ip addr show eth0 | grep inet

# Уровень 4: Транспортный
netstat -tulpn | grep LISTEN

# Уровень 5: Сессионный
netstat -an | grep ESTABLISHED

# Уровень 6: Представленческий
curl -v https://google.com 2>&1 | grep "SSL certificate"

# Уровень 7: Прикладной
lsof -i | grep LISTEN
```

## 🛠️ Команды для проверки каждого уровня (Windows PowerShell)

```powershell
# Уровень 1: Физический
Get-NetAdapter | Select-Object Name, LinkSpeed, LinkStatus

# Уровень 2: Канальный
Get-NetAdapter -Name "Ethernet" | Select-Object MacAddress

# Уровень 3: Сетевой
Get-NetIPAddress -AddressFamily IPv4 | Select-Object IPAddress

# Уровень 4: Транспортный
Get-NetTCPConnection -State Listen | Select-Object LocalPort, OwningProcess

# Уровень 5: Сессионный
Get-NetTCPConnection -State Established | Select-Object RemoteAddress

# Уровень 6: Представленческий
curl -v https://google.com 2>&1 | Select-String "SSL"

# Уровень 7: Прикладной
lsof -i | Select-String LISTEN
```

## 🎯 Типичные проблемы и их решения (по уровням)

### Проблема: "Нет интернета!"

| Уровень | Причина | Решение |
|---------|---------|---------|
| **1** | Кабель не подключён | Вставь кабель или включи Wi-Fi 🔌 |
| **2** | Неправильный MAC-фильтр | Отключи фильтрацию в роутере 📮 |
| **3** | Нет маршрута по умолчанию | Проверь `ip route show` 🏠 |
| **4** | Порт заблокирован | Открой порт 80/443 на фаерволе ✂️ |
| **5** | Не установилось соединение | Попробуй другой сайт или время 📞 |
| **6** | Сертификат не верен | Обнови браузер или SSL-сертификаты 🔒 |
| **7** | Браузер устарел | Обновись до последней версии 🌐 |

## 🎬 Полный пример: Как работает запрос к сайту (по уровням!)

```
1. Ты вводишь google.com → Уровень 7 (браузер)
2. Запрос шифруется HTTPS → Уровень 6 (шифрование)
3. Устанавливается соединение → Уровень 5 (сессионный)
4. Данные разбиваются на пакеты → Уровень 4 (транспортный)
5. Каждому пакету присваивается IP → Уровень 3 (сетевой)
6. Пакеты адресуются по MAC → Уровень 2 (канальный)
7. По кабелю/Wi-Fi летят сигналы → Уровень 1 (физический)

Ответ идёт в обратном порядке! 🔄
```
