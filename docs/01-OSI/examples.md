### Кейс 1: "Не работает Wi-Fi" 🔴

**Симптомы:** Компьютер не видит сеть, но кабель работает.

**Анализ по уровням OSI:**

| Уровень | Проверка | Результат |
|---------|----------|-----------|
| 7 (Application) | Браузер открывает сайты? | ✅ Да |
| 6-5 (Presentation/Session) | HTTPS работает? | ✅ Да |
| 4 (Transport) | TCP соединения есть? | ✅ Да |
| 3 (Network) | IP адрес получен? | ✅ Да |
| 2 (Data Link) | MAC-адрес виден? | ❌ Нет |
| 1 (Physical) | Wi-Fi адаптер активен? | ❌ Отключён |

**Решение:** Проблема на физическом уровне — Wi-Fi адаптер отключён в системе!

```bash
# Проверка состояния Wi-Fi интерфейса
sudo ip link show wlan0
# Если состояние DOWN — нужно включить: sudo ip link set wlan0 up
```

### Кейс 2: "Медленно загружаются сайты" 🐌

**Симптомы:** Сайты открываются, но очень медленно.

**Анализ по уровням OSI:**

| Уровень | Проблема | Решение |
|---------|----------|---------|
| 4 (Transport) | TCP slow start слишком медленный | Настроить TCP tuning |
| 3 (Network) | Маршруты через много прыжков | Оптимизировать маршрутизацию |
| 2 (Data Link) | Высокий коллизии на канале | Увеличить скорость канала |

**Решение:** Проблема на транспортном уровне — TCP настройки.

```bash
# Проверка TCP настроек Linux
sysctl net.ipv4.tcp_tw_reuse
sysctl net.ipv4.tcp_fin_timeout
```

---

### Кейс 3: "Не могу подключиться к принтеру" 🖨️

**Симптомы:** Принтер виден в сети, но файлы не передаются.

**Анализ по уровням OSI:**

| Уровень | Проблема | Решение |
|---------|----------|---------|
| 7 (Application) | Служба печати отключена | Включить службу CUPS |
| 4 (Transport) | Порт 9100 закрыт фаерволом | Открыть порт в iptables |

**Решение:** Проблема на прикладном уровне — служба печати.

```bash
# Проверка статуса службы печати
systemctl status cups
# Включение: sudo systemctl enable --now cups
```

---

### Кейс 4: "Не работает видеозвонок" 📹

**Симптомы:** Аудио есть, но видео не идёт.

**Анализ по уровням OSI:**

| Уровень | Проблема | Решение |
|---------|----------|---------|
| 6 (Presentation) | Кодеки видео не поддерживаются | Установить кодеки |
| 4 (Transport) | UDP пакеты теряются | Настроить QoS для RTP |

**Решение:** Проблема на представленческом уровне — кодеки.

```bash
# Установка кодеков для видеозвонков
sudo apt install libavcodec-extra
```

## 🛠️ Инструменты диагностики по уровням OSI

### Уровень 1-2: Физический и Канальный

```bash
# Проверка физического соединения
ethtool eth0          # Состояние Ethernet
iwconfig              # Wi-Fi информация
dmesg | grep -i eth    # Логи драйвера

# Просмотр MAC-адресов
arp -a                # Таблица ARP
ip neigh show         # ARP таблица Linux
```

### Уровень 3: Сетевой

```bash
# IP конфигурация
ip addr show          # Все интерфейсы и IP
ifconfig              # Устаревший, но работает

# Маршрутизация
route -n              # Таблица маршрутов
ip route show         # Современная альтернатива

# ICMP (ping)
ping -c 4 google.com  # Проверка доступности
```

### Уровень 4: Транспортный

```bash
# Порты и соединения
netstat -tulpn        # Все порты и процессы
ss -tulpn             # Современная альтернатива netstat

# Просмотр TCP/UDP статистики
cat /proc/net/tcp     # TCP соединения (hex)
cat /proc/net/udp     # UDP соединения
```

### Уровень 5-7: Сессионный, Представленческий, Прикладной

```bash
# HTTP/HTTPS запросы
curl -v https://google.com    # Подробные логи
wget --verbose https://example.com

# DNS (часть прикладного)
nslookup google.com           # Проверка DNS
dig google.com                # Детальная информация
host google.com               # Альтернатива nslookup

# Почта (SMTP/POP3/IMAP)
telnet mail.example.com 25  # SMTP порт
```
