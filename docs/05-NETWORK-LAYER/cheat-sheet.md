# 🌐 Сетевой слой (Layer 3) — Шпаргалка

## 📊 Основные понятия

| Понятие | Определение | Пример |
|---------|-------------|--------|
| **IP-адрес** | Логический адрес устройства | `192.168.1.100` |
| **Маршрутизатор** | Устройство для направления пакетов между сетями | Домашний роутер |
| **Шлюз (Gateway)** | Выход из локальной сети в другую сеть | Маршрутизатор дома |
| **Подсеть** | Часть большой сети | `192.168.1.0/24` |
| **Маска подсети** | Определяет границы подсети | `/24`, `255.255.255.0` |

---

## 📊 Классы IP-адресов (IPv4)

| Класс | Диапазон первого октета | Пример | Назначение |
|-------|------------------------|--------|------------|
| **A** | 1-126 | `10.0.0.1` | Крупные сети |
| **B** | 128-191 | `172.16.0.1` | Средние сети |
| **C** | 192-223 | `192.168.1.1` | Маленькие сети |

| Класс | Диапазон | Примеры |

| B | 172.16.0.0 - 172.31.255.255 | `172.16.0.1` |

## 🔄 Основные команды для сетевого слоя

### **Linux/Mac:**
```bash
# Показать все сетевые интерфейсы
ip addr show

# Или (старый способ):
ifconfig -a

# Показать таблицу маршрутов
ip route show

# Или:
route -n

# Проверить DNS серверы
cat /etc/resolv.conf

# Тестирование соединения
ping 8.8.8.8

# Просмотр открытых портов
netstat -tlnp

# Или (современнее):
ss -tlnp
```

### **Windows:**
```cmd
# Показать все сетевые интерфейсы
ipconfig /all

# Показать таблицу маршрутов
route print

# Проверить DNS серверы
nslookup google.com

# Тестирование соединения
ping 8.8.8.8

# Просмотр открытых портов
netstat -ano
```

## 📋 ICMP — команды диагностики

### **Ping** — проверка доступности хоста:
```bash
# Linux/Mac:
ping -c 4 8.8.8.8

# Windows:
ping 8.8.8.8
```

### **Traceroute** — отслеживание пути пакета:
```bash
# Linux/Mac:
traceroute google.com

# Windows:
tracert google.com
```

### **Mtr** — комбинация ping и traceroute:
```bash
# Linux/Mac:
mtr google.com -c 20

# Windows:
pathping google.com
```

## 🌐 IPv6 — сокращение адресов

### **Правила сокращения:**
1. Убрать ведущие нули в каждой группе
2. Заменить последовательность групп `0000` на `::` (только один раз!)

**Примеры:**
```
Полный: 2001:0db8:85a3:0000:0000:8a2e:0370:7334
Сокращённый: 2001:db8:85a3::8a2e:370:7334

Полный: fe80:0000:0000:0000:0000:0000:0000:0001
Сокращённый: fe80::1
```

## 🔍 Быстрая диагностика проблем

### **Нет доступа в интернет:**
```bash
# 1. Проверь IP-адрес
ip addr show | grep "inet"

# 2. Проверь шлюз
ip route show | grep default

# 3. Проверь DNS
cat /etc/resolv.conf

# 4. Тестируй доступность шлюза
ping <шлюз>
```

### **Высокий пинг и потери пакетов:**
```bash
# Проверь качество соединения с шлюзом
ping -c 100 <шлюз>

# Отслежь путь до назначения
traceroute google.com

# Проверь на наличие проблем с сетевым кабелем/Wi-Fi
sudo ethtool eth0 | grep -i errors
```

### **Не работает DNS:**
```bash
# Проверь, что DNS серверы настроены
cat /etc/resolv.conf

# Тестируй доступность DNS сервера
nslookup google.com 8.8.8.8

# Попробуй другие DNS серверы
nslookup google.com 1.1.1.1
```

## 📊 Сравнение IPv4 и IPv6

| Характеристика | IPv4 | IPv6 |
|----------------|------|------|
| Размер адреса | 32 бита | 128 бит |
| Формат | Десятичный (X.X.X.X) | Шестнадцатеричный (XXXX:...) |
| Количество адресов | ~4,3 млрд | 3,4×10³⁸ |
| Подсети | Классы A/B/C | Предфиксы /64, /48 и т.д. |
| Автоматическая настройка | DHCP | SLAAC |
| Безопасность | Отдельный протокол IPsec | Встроен в стек |
