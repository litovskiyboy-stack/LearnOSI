# 🌐 Сетевой слой (Layer 3) — Основы для начинающих

## 📚 Что изучаем в этом разделе?

Сетевой слой отвечает за **логическую адресацию** и **маршрутизацию** данных между сетями. Это "почта" интернета, которая доставляет пакеты от отправителя к получателю через множество промежуточных узлов.

---

## 🎯 Ключевые понятия

### **IP-адрес (Internet Protocol)**
Логический адрес устройства в сети, как почтовый индекс для дома.

**Пример:** `192.168.1.100` — это IP-адрес твоего компьютера в локальной сети.

---

### **Маршрутизатор (Router)**
Устройство, которое направляет пакеты между сетями.

**Как работает маршрутизатор:**
```
Компьютер → Маршрутизатор → Интернет → Сервер
           ↓
    Принимает пакет и решает: "Куда отправить дальше?"
```

---

### **Шлюз (Gateway)**
Устройство, через которое компьютер выходит в другую сеть.

**Пример:** Твой домашний маршрутизатор — это шлюз для выхода в интернет.

---

## 📊 Типы IP-адресов

### **IPv4** — Классическая система (32 бита)
Формат: `XXX.XXX.XXX.XXX` (4 числа от 0 до 255)

**Примеры:**
- `192.168.1.100` — частный адрес (локальная сеть)
- `8.8.8.8` — публичный адрес (DNS Google)
- `172.16.50.1` — частный адрес

**Всего адресов:** 4,3 миллиарда (~4,3 млрд)

---

### **IPv6** — Новая система (128 бит)
Формат: `XXXX:XXXX:XXXX:...` (8 групп по 4 шестнадцатеричных цифры)

**Примеры:**
- `2001:0db8:85a3:0000:0000:8a2e:0370:7334`
- `fe80::1` — локальный адрес

**Всего адресов:** 3,4×10³⁸ (невообразимо много!)

---

## 🏠 Классы IP-адресов (IPv4)

### **Class A** — Для огромных сетей
- Формат: `A.B.C.D`
- Диапазон первого октета: 1-126
- Пример: `10.0.0.1`, `172.15.0.1`

### **Class B** — Для средних сетей
- Формат: `A.B.C.D`  
- Диапазон первого октета: 128-191
- Пример: `172.16.0.1`, `192.168.1.1`

### **Class C** — Для маленьких сетей
- Формат: `A.B.C.D`
- Диапазон первого октета: 192-223
- Пример: `192.168.0.1`, `192.168.1.100`

---

## 🔢 Частные и публичные адреса

### **Частные (Private) адреса** — Только внутри локальной сети

Не маршрутизируются в интернете, можно использовать повторно:

| Класс | Диапазон | Примеры |
|-------|----------|---------|
| A | 10.0.0.0 - 10.255.255.255 | `10.0.0.1` |
| B | 172.16.0.0 - 172.31.255.255 | `172.16.0.1` |
| C | 192.168.0.0 - 192.168.255.255 | `192.168.1.1` |

---

### **Публичные (Public) адреса** — Видны в интернете

Уникальные адреса, которые можно использовать глобально:
- `8.8.8.8` — DNS Google
- `1.1.1.1` — DNS Cloudflare
- `208.67.222.222` — DNS OpenDNS

---

## 🧮 Подсети (Subnets)

### **Что такое подсеть?**
Разделение большой сети на меньшие части для лучшей организации и безопасности.

**Пример:**
```
Большая сеть: 192.168.1.0/24
├─ Подсеть 1: 192.168.1.0/26 (62 адреса)
├─ Подсеть 2: 192.168.1.64/26 (62 адреса)  
└─ Подсеть 3: 192.168.1.128/26 (62 адреса)
```

---

### **Маска подсети** — Определяет, какие биты адреса — сеть, а какие — хост

| Маска | Двоично | Количество хостов |
|-------|---------|-------------------|
| /24 | 11111111.11111111.11111111.00000000 | 254 |
| /26 | 11111111.11111111.11111111.11000000 | 62 |
| /30 | 11111111.11111111.11111111.11110000 | 2 |

---

## 🔄 Как работает маршрутизация?

### **Простой пример:**
```
Твой компьютер (192.168.1.100) хочет отправить данные в Google (8.8.8.8):

1. Компьютер проверяет: "Это локальная сеть или нет?"
   → Нет, 8.8.8.8 не в моей подсети
   
2. Отправляет пакет на шлюз (маршрутизатор)
   
3. Маршрутизатор смотрит таблицу маршрутов:
   "Куда отправить пакеты для Google? → Через ISP"
   
4. Пакет идёт через интернет к DNS серверу Google
   
5. DNS возвращает IP-адрес Google, и данные доходят!
```

## 📋 Основные команды для работы с сетевым слоем

### **Linux/Mac:**
```bash
# Показать все сетевые интерфейсы
ip addr show

# Или (старый способ):
ifconfig -a

# Показать таблицу маршрутов
ip route show

# Или:
route -n

# Проверить DNS серверы
cat /etc/resolv.conf

# Тестирование соединения
ping 8.8.8.8

# Просмотр открытых портов
netstat -tlnp

# Или (современнее):
ss -tlnp
```

### **Windows:**
```cmd
# Показать все сетевые интерфейсы
ipconfig /all

# Показать таблицу маршрутов
route print

# Проверить DNS серверы
nslookup google.com

# Тестирование соединения
ping 8.8.8.8

# Просмотр открытых портов
netstat -ano
```

## 🎯 Практическое задание: Проверка своей сети

### **Шаг 1. Узнай свой IP-адрес**

#### **Linux/Mac:**
```bash
ip addr show | grep "inet"
```

Ожидаемый результат:
```
inet 192.168.1.100/24 brd 192.168.1.255 scope global eth0
```

#### **Windows:**
```cmd
ipconfig /all
```

Ожидаемый результат:
```
IPv4 Address. . . . . . . . . . . : 192.168.1.100
Subnet Mask . . . . . . . . . . . : 255.255.255.0
Default Gateway. . . . . . . . . . : 192.168.1.1
```

---

### **Шаг 2. Узнай свой шлюз (Gateway)**

#### **Linux/Mac:**
```bash
ip route show | grep default
```

Ожидаемый результат:
```
default via 192.168.1.1 dev eth0
```

#### **Windows:**
```cmd
route print
```

Ищи строку "Interface" с твоим IP и "Gateway".

---

### **Шаг 3. Узнай свои DNS серверы**

#### **Linux/Mac:**
```bash
cat /etc/resolv.conf
```

Ожидаемый результат:
```
nameserver 8.8.8.8
nameserver 8.8.4.4
```

#### **Windows:**
```cmd
ipconfig /all
```

Ищи строку "DNS Servers".

---

### **Шаг 4. Проверь соединение с Google DNS**

```bash
ping -c 4 8.8.8.8    # Linux/Mac
ping 8.8.8.8         # Windows
```

Ожидаемый результат:
```
PING 8.8.8.8 (8.8.8.8) 56(84) bytes of data.
64 bytes from 8.8.8.8: icmp_seq=1 ttl=114 time=2.3 ms
64 bytes from 8.8.8.8: icmp_seq=2 ttl=114 time=2.1 ms
...

--- 100% packet loss
```
